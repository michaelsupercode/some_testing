GIT VERTIEFUNG

  1. Erstelle ein Projekt & eine Repo
  2. Lade das Team ein
  3. Nun erstellst du eine Branch
    • git branch feature2 -> legt einen neuen branch an.
    • git checkout feature2 -> wechselt zum branch feature2.
    • git checkout -b feature2 -> legt einen neuen branch und wechselt direkt dorthin
    • git branch -d feature2 -> löscht den branch feature2.
    • git branch -D feature2 -> Branch wird unabhängig von seinem Status und ohne Warnung gelöscht.
    • git branch -> zeigt alle vorhandenen branches an. (\* -> zeigt an, wo man sich gerade befindet)

Commits

  • git log --all: zeigt dir alle commits an
  • git log --oneline: alle commits in einer Zeile
  • git log --graph: zeigt dir alle commits grafisch an
  • mit "q" kommst du aus der Ansicht wieder raus

Das Mergen

Sobald sämtliche Teams ihre Änderungen erfolgreich in den main Branch eingebracht haben und dieser auf dem neuesten Stand ist, ist es nun an der Zeit, die Inhalte des Main Branches in unseren eigenen Zweig zu mergen

  1. Falls du dich nicht in deiner Branch befindest -> git checkout feature2
  2. git merge main -> main wird mit der branch gemerged
  3. git checkout main
  4. lösche die Branch, wenn du sie nicht mehr benötigst -> git branch -d feature2

Commit zurück gehen und Änderungen übernehmen

  1. du kannst git log --oneline verwenden, um die Liste der Commit-IDs anzuzeigen
    • die id auswählen, die man haben möchte -> Vorraussätzung ist, dass du regelmäßig pusht

  2. Du möchtest aus einem bestimmten Commit einen neuen Branch erstellen und auf diesen wechseln.
    • git checkout -b "neuer-branch-name" "commit-id"

  3. git add .
  4. git commit -m "back"
  5. git push origin "neuer-branch-name"
