GitHub GIT CHEAT SHEET

Git ist das freie und quelloffene verteilte Versionskontrollsystem, das für alles verantwortlich ist, was mit GitHub zu tun hat, das lokal auf Ihrem Computer stattfindet. Dieser Spickzettel enthält die wichtigsten und am häufigsten Git-Befehle zum einfachen Nachschlagen.

Installation & Graphical User Interfaces

Mit plattformspezifischen Installationsprogrammen für Git bietet GitHub auch die die Möglichkeit, mit den neuesten Versionen des Kommandozeilen-Tools auf dem Laufenden zu bleiben und gleichzeitig eine grafische Benutzeroberfläche für die tägliche Interaktion, Überprüfung und Repository-Synchronisierung.

Einrichtung

Konfigurieren von Benutzerinformationen, die in allen lokalen Repositories verwendet werden

git config --global user.name “[firstname lastname]”

einen Namen festzulegen, der bei der Überprüfung des Versionsverlaufs für die Anerkennung identifizierbar ist

git config --global user.email “[valid-email]”

eine E-Mail-Adresse festlegen, die mit jeder Verlaufsmarkierung verknüpft werden soll

git config --global color.ui auto

automatische Einfärbung der Befehlszeile für Git zur leichteren Überprüfung einstellen

Einrichtung & Inbetriebnahme

Konfigurieren von Benutzerinformationen, Initialisieren und Klonen von Repositories

git init

initialize an existing directory as a Git repository

git clone [url]

retrieve an entire repository from a hosted location via URL

Etappe & Schnappschuss

Arbeiten mit Snapshots und dem Git-Speicherbereich

git status

geänderte Dateien im Arbeitsverzeichnis anzeigen, die für die nächste Übertragung bereitgestellt werden

git add [file]

fügt eine Datei so, wie sie jetzt aussieht, zu Ihrer nächsten Übertragung hinzu (stage)

git reset [file]

eine Datei zurücksetzen und dabei die Änderungen im Arbeitsverzeichnis beibehalten

git diff

Diff der geänderten, aber nicht ausgelagerten Dateien

git diff --staged

diff der Änderungen, die bereitgestellt, aber noch nicht übertragen wurden

git commit -m “[descriptive message]”

den bereitgestellten Inhalt als neuen Commit-Snapshot übertragen

Verzweigen & Zusammenführen

Arbeiten mit Snapshots und dem Git-Speicherbereich

git branch

listet Ihre Zweige auf. Neben dem derzeit aktiven Zweig wird ein * angezeigt

git branch [branch-name]

erstellt einen neuen Zweig mit dem aktuellen Commit

git checkout

wechselt zu einem anderen Zweig und checkt ihn in Ihr Arbeitsverzeichnis aus

git merge [branch]

fügt die Historie des angegebenen Zweigs in den aktuellen Zweig ein

git log

zeigt alle Übertragungen in der Historie des aktuellen Zweigs an

Prüfen & Vergleichen

Prüfung von Protokollen, Diffs und Objektinformationen

git log

zeigt den Verlauf der Übertragungen für den aktuellen Zweig an

git log branchB..branchA

zeigt die Übertragungen in ZweigA an, die sich nicht in ZweigB befinden

git log --follow [file]

zeigt die Übertragungen an, die die Datei geändert haben, auch über Umbenennungen hinweg

git diff branchB...branchA

zeigt den Unterschied zwischen BranchA und BranchB an

git show [SHA]

zeigt jedes Objekt in Git in menschenlesbarem Format an

Teilen & Aktualisieren

Abrufen von Aktualisierungen aus einem anderen Repository und Aktualisieren lokaler Repos

git log

Hinzufügen einer Git-URL als Alias

git log branchB..branchA

holt alle Zweige von diesem Git-Remote ab

git log --follow [file]

einen entfernten Zweig mit dem aktuellen Zweig zusammenführen, um ihn auf den neuesten Stand zu bringen

git diff branchB...branchA

Übertragen von Übertragungen des lokalen Zweigs an den Zweig des entfernten Repositorys

git show [SHA]

holt alle Übertragungen aus dem entfernten Zweig und führt sie zusammen
